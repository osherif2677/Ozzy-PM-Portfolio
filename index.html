Excel, SQL, Python & R – how I actually use them

Tools are only useful if they help me make better decisions. Here’s how I typically use each one to support the programs on this page.

Excel – control sheets & KPI rules
Excel SUMIFS($INF_Pct,$Month,">="&$Start_Q1,$Month,"<="&$End_Q1) =IF([@HP_Agile]<0.90,"RED", IF([@HP_Agile]<0.95,"YELLOW","GREEN")) =AVERAGEIFS($Shrink_Pct,$Month,">="&$Start_Jan,$Month,"<="&$End_Apr)
SQL – aggregating INF by department & month
SQL SELECT site_id, month, dept, SUM(inf_units) AS inf_units, SUM(requested_units) AS requested_units, SUM(inf_units) * 1.0 / NULLIF(SUM(requested_units), 0) AS inf_pct FROM outbound_inf_fact WHERE site_id = 'MAG5' AND month BETWEEN '2025-01-01' AND '2025-11-01' GROUP BY site_id, month, dept ORDER BY month, inf_pct DESC;
Python (pandas) – quick groupby views
Python import pandas as pd kpi_df = pd.read_csv("mag5_kpi_quarterly.csv") summary = ( kpi_df .groupby("quarter") .agg({ "hp_agile": "mean", "produce_culling": "mean", "expiration_compliance": "mean", "total_shrink_pct": "mean" }) .round(3) ) print(summary.loc[["Q1","Q4"]])
R – simple INF trend visualization
R library(ggplot2) ggplot(inf_df, aes(x = month, y = inf_pct, color = dept)) + geom_line(size = 1) + labs(title = "MAG5 Outbound INF% by department", x = "Month", y = "INF %") + theme_minimal()

Acronym glossary (for non-Fresh readers)

INF
Item shows as in-stock in the system but cannot be found at pick – usually driven by inventory accuracy, restock latency, or location errors.
SDU
Item becomes unavailable for same-day orders due to inventory, substitution, or operational constraints.
HP Agile
Store-level inventory execution metric (Amazon internal) covering key routines and standard work.
Gaps-with-stock
Out-of-stock on the shelf while backroom still has inventory – a classic execution issue.
OPS / shipped OPS
Operational sales / shipped operational sales – used as the denominator for shrink %.
